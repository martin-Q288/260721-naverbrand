# 네이버 브랜드 커넥트 블로그 원고 파이프라인

제품 URL과 상세페이지 스크린샷을 넣으면, 마스터 프롬프트 규칙(`CLAUDE.md`)대로
**기획안·완성 원고**를 자동으로 만들어 주는 도구입니다.

두 가지 방식으로 쓸 수 있습니다.

- **웹앱(`blog.html`)** — 브라우저에서 바로 사용. 설치·터미널 불필요.
- **Claude Code 프로젝트** — 폴더를 열고 채팅으로 요청 (원래 방식).

---

## A. 웹앱으로 쓰기 (`blog.html`)

브라우저에서 Anthropic Claude API를 직접 호출하는 **단일 파일 웹앱**입니다.

### 사용법
1. `blog.html`을 브라우저로 엽니다. (또는 아래 GitHub Pages 주소로 접속)
2. 우측 상단 **API 설정**에서 Anthropic API 키(`sk-ant-...`)를 입력합니다.
   - 키는 이 브라우저의 `localStorage`에만 저장되며 외부로 전송되지 않습니다(API 호출 제외).
3. 입력 후 **원고 생성**:
   - **제품 URL** 붙여넣기
   - **상세페이지 스크린샷** 업로드 (제품명·평점·리뷰수·가격·스펙 화면) — 여러 장 가능
   - **모드**: 기획안 / 완성 원고
   - **시간대 톤**: 08시(정보성) / 12시(비교·체크리스트) / 18시(스토리·CTA 강)
   - (선택) **실제 판매가·할인** → "지금 사야 하는 이유" 섹션 반영
   - (선택) **오늘 이미 쓴 제목** → 중복 방지
4. 결과를 **복사** 하거나 **`.md`로 저장**합니다.

### GitHub Pages로 호스팅 (무료)
저장소가 **Public**이면:
1. GitHub 저장소 → **Settings → Pages**
2. Source: **Deploy from a branch**, Branch: `main` / `/(root)` → **Save**
3. 잠시 후 접속: `https://martin-q288.github.io/260721-naverbrand/blog.html`

### 주의
- 이 앱은 브라우저에서 API 키를 직접 사용합니다. **본인/신뢰된 사람만** 쓰는 용도로 적합합니다.
  불특정 다수에게 공개 배포하려면 키를 감추는 서버 프록시를 앞에 두세요.
- 네이버 페이지를 **자동 크롤링하지 않습니다.** 평점·리뷰수·상세는 스크린샷에서만 읽고,
  스펙·경쟁가 등 공개 정보만 웹 검색으로 조사합니다.

## B. Claude Code 프로젝트로 쓰기 (원래 방식)

1. **Claude Code 데스크톱 앱**을 설치하고 구독 계정으로 로그인합니다.
   - 공식 문서: https://code.claude.com/docs/ko/setup
2. 앱에서 이 폴더를 **"폴더 열기"**로 엽니다.
3. 상세페이지 스크린샷을 준비해 두고, 채팅창에 이렇게 요청합니다.
   ```
   이 URL 제품으로 12시 기획안 써줘.
   URL: (제품 링크 붙여넣기)
   ```
   - **기획안 먼저** → 방향 확인 후 **"이 방향으로 완성 원고 써줘"**
   - **할인 반영**: "판매가 119만원, 20% 할인이야"
   - **중복 방지**: "오늘 이미 쓴 제목: OOO"

전체 규칙은 `CLAUDE.md`에 있습니다.
